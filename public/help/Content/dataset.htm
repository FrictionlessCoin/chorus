Working with your data sources

The Dataset detail tab displays details about a particular dataset.

Features of the Datasets page

  • Creating a sandbox from the Data tab
  • Exploring the Data tab (basic tasks)
  • Working with source datasets and Chorus views

Creating a sandbox from the Data tab

Create a sandbox so that you can access the full feature set associated with the Data tab:

  1. Hover over and then select the "?" icon next to Import File.
  2. Using drop-down options, you can select an instance, database, and schema to associate with your sandbox. Each workspace can only be associated with a single sandbox.

Benefits of working with a sandbox

When you create a sandbox, you can then easily import interesting files or datasets from other schemas into your sandbox as your workspace's copy that you are then free to manipulate and transform.

Exploring the Data tab (basic tasks)

If you do not have a sandbox, you can only perform the following basic functions when browsing a data tab:

  • Add a note
  • Preview Data

You can however, still visualize data and create Chorus Views from datasets that have been associated with the workspace.

Working with source datasets and Chorus views

You can perform the following additional tasks when working with source datasets and Chorus views for a workspace that has an associated sandbox.

  • Import Now: This option functions only on source tables and views, and on Chorus views. Use it to immediately import a table into a new or existing one. You can only perform this once. You can specify the number that are being imported. If can import the data into an existing sandbox table. You can truncate the data. You must specify the table name from the drop-down list.
  • Create import schedule: This automatically imports data into a new or existing table, limit the number of rows, and schedule a start and end time. You can set up a daily, weekly, or monthly schedule.
