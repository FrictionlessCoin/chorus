Users

The Users page displays a complete list of Chorus users.

Features of the Users page

  • Permissions Model
  • Viewing the list of users
  • Administering users (Administrators only)

Permissions Model

Currently Chorus supports a simple permissions model and can only provision users and administrator accounts.

A user is a member of Chorus and can own objects.

You can configure Chorus to use LDAP protocol for all users whereby username and password are authenticated by the LDAP server, or you can configure Chorus to manage all passwords locally. See the Chorus Installation Guide for details.

Viewing the list of users

  • Explore the users in the list, sorted by last name. The list calls out Chorus administrators.
  • Change the default to sort by first name.
  • Select a username to go to their page for more information. See "Users".

Administering users (Administrators only)

Only an administrator can add or delete a user. Administrators can also give users Administrator permissions. Multiple administrators can exist within Chorus.
