Work Files

Work Files is the third tab in the workspace folder. The Work Files tab displays the work file list page. Your work file draft is auto-saved every 30 seconds.

Features of the Work Files tab

Viewing the list of work files.

Editing work files (all users)

Editing work files (Administrators or owners only)

Viewing the list of Work Files

  • Displays a list of all the work files associated with a workspace, the default view is Show All File. The other drop down options you can select are as follows:
  • SQL files
  • Code files
  • Text files
  • Other files
  • The work files are sorted Alphabetically, by default. But you can also sort By Date.

Editing work files (only members of a workspace)

The following is a list of tasks that all users can perform.

  • Copy latest version to another workspace: Brings up the Copy File to Workspace displaying a list of the workspaces that you own. Note that the configurable limit is 250 MB. When you set # Maximum upload work file size, you can use 'M', or 'G' as a short cut to specify megabyte and gigabyte in the chorus.properties file. For example: chorus.workfile.max_file_size=250M.
  • Download: Download any work files that you or another member of your workspace has uploaded.
  • Add a note: Add a note to comment on a particular work file.You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.

Editing work files (Administrators or owners only)

The following is an list of additional tasks that you can perform if you own the workspace or are a Chorus Administrator.

  • Upload File to upload a file to your workspace.
  • Create SQL File to create a blank SQL file. You can also upload your own SQL file.
  • Delete: the file from your workspace.
