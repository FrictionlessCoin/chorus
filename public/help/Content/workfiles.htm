Work Files

Work Files is the third tab in the workspace. The Work Files tab displays the work file list page. Your work file draft is auto-saved every 30 seconds.

Features of the Work Files tab

  • Viewing the list of work files.
  • Editing work files (only members of a workspace)
  • Editing work files (Administrators or owners only)

Viewing the list of Work Files

  • Displays a list of all the work files associated with a workspace, the default view is Show All File. The other drop down options you can select are as follows:
  • SQL files
  • Code files
  • Text files
  • Other files
  • The work files are sorted Alphabetically, by default. But you can also sort By Date.

Editing work files (workspace members)

The following is a list of tasks that all workspace members can perform.

* these tasks can be performed by non-workspace members.

  • *Copy latest version to another workspace: Brings up the Copy File to Workspace displaying a list of the workspaces that you own.
  • *Download: Download any work files that you or another member of your workspace has uploaded.
  • *Add a note: Add a note to comment on a particular work file.You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.
  • Save a new version of a work file. You can edit a work file and save as a new version, and also have the option to add version notes when save the new work file.
  • Add Functions and Datasets. You can add functions (with their arguments) or datasets (table/view names and column names) to work files without having to navigate around to locate them. To add a function or dataset, open the work file, then select the function or dataset sub-tab in the right pane, and click on the appropriate option.
  • Save as draft. You can save your SQL work file as a draft to be worked on later.
  • Execute the SQL file. You can execute an SQL work file in any of the following ways:
    • Execute the whole work file. Click the Run button in the workspace sandbox. The shortcuts for this option are CRTL+r (Mac) and ALT+r (Windows).
    • Execute selected SQL. You can execute part of your SQL statements in the sandbox. The shortcuts for this option are CRTL+e (Mac) and ALT+e (Windows).
    • Run in another schema. You can execute the work file against a different schema.
  • Upload File: Allows you to upload a file to your workspace.
  • Create SQL File: Allows you to create a blank SQL file. You can also upload your own SQL file.
  • Delete: Presents the option to delete the file from your workspace.
