Instances

This topic should open from the context-sensitive linkinstance.

The Instances page displays a list registered Greenplum and other instances. It displays the total number of and provides you with the ability to add a new instance. It also shows the activity stream of the highlighted instance. Instances are set up during installation .

Features of the Instances page

  • Displays the total number of instances.
  • Splits display of into registered Greenplum and Hadoop instances.
  • Select Add Instance to register a new instance. Instances are only set up during installation.
  • Highlight an instance to select it for editing.
  • Displays recent activity and configuration information.

Editing an instance

Select the instance from the left pane. You will see the instance name in the right pane, with information including the recent activity associated with the instance. You can select from the following links to perform the following tasks:

Edit: This allows you to modify and manage instance-wide permissions. When selected, it presents the Instance Permissions dialog that displays the owner, you can remove shared permissions for this instance, edit the username and password, and change ownership.

Edit Instance: This presents the Edit instance dialog. This dialog requires the following information:

  • Instance Name: You may want to edit the name of an instance for a variety of reasons, such as <placeholder for information from developers here>.
  • Description: Shows URL in the samples.
  • Host:Shows URL of host.
  • Port: Port number of host.
  • Database Name:Type of database, Postgres, Oracle, etc...

View Workspaces: Shows the number of workspaces that you can view. Presents the Chorus Usage by Workspace dialog. This dialog shows information about the location and the owner of workspaces.

Add a note: Add a note to comment on a particular instance.You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.
