Data

Data is the second tab in the workspace. The Data tab displays the list of datasets associated with the workspace that you have chosen to explore.

The Data tab is accessible even if you do not have a sandbox. If you have not created a sandbox, the Data tab will not be populated and the import file icon is disabled. See below for information about adding a sandbox.

If you do not have a sandbox, you can still associated datasets by browsing the instances that have been registered. To do this, drill down to a dataset from an instance, then associate a dataset to a workspace. See Instances for more details.

Note: If you associate datasets from instances when there is no sandbox, you will not be able to import files into the sandbox or use any of the import schedule options.

To add a sandbox, See "Adding a Sandbox" .

Features of the Data tab

  • Viewing datasets
  • Exploring a particular dataset
  • Importing data

Viewing datasets

  •  Browse the list of associated datasets in the left pane. The Data tab defaults to the Show all datasets: option. Selecting one of the other viewing options from the drop-down list determines how you can work with the datasets available. These options are as follows:
    • source datasets: Source datasets are datasets that have been associated with the workspace but are not materialized in the workspace's sandbox. Source datasets can be imported into the workspace's schema.
    • Chorus Views: A Chorus view is a select statement/dataset that has not materialized in the workspace's sandbox. Once you have selected a Chorus view, you can it edit by clicking the Edit button, which will open up the code mirror interface (editor). Once you have made your edits, select run. You can cancel your changes, or save and return to column view mode.
    • sandbox datasets: The workspace's sandbox schema.
  • Select a particular dataset to view details. See "Working with your data sources" for more information.

Exploring a particular data source

Select the dataset that you wish to explore. This brings you to the details page of the dataset.

  • Explore: Select this option to search for columns within the dataset.
  • Derive: Select this option to create a Chorus view. You can preview your SQL statement, add a join and select columns.
    • You can derive Chorus views from source datasets or sandbox datasets. Chorus views can only be created when viewing the dataset from a workspace, not from the instance browser.
    • Click duplicate this chorus view link from an already opened Chorus view to make a copy of a Chorus view.
    • Click create as a database view link from an already opened Chorus view to convert into a database view.
    • Click the Edit button from an already opened Chorus view to edit the SQL of the Chorus view.
  • Add a note: Select this option to add a note to a dataset in a schema. If you are working from the workspace, you can support your note with attachments from the desktop, datasets, or work files. If you are working from the instance browser you can only support your note with attachments from the desktop.
  • Preview data: Select this option to preview a table's content and data. This queries the table and returns the first 100 rows in a table. You also have the option to download to your desktop a specific number of rows, or to download all the rows.
  • Run analyze Select this option to collect statistics about the contents of a table. The results of this analyze are stored in the system table pg_statisic. The query planner will subsequently use these statistics to help determine the most efficient execution plans for queries. These statistics, including row count, common values, and so on, can be viewed from the Information sub-tab for the table. Note: Running analyze on large tables can significantly slow down queries, so should be used with caution.
  • Visualize: Select this option to create a visual representation based on your data. You can also add data filters. The types of charts you can use to visualize your data are as follows:
  • chart
  • Histogram
  • Heatmap
  • Time Series
  • Boxplot

Importing Data

You can only run imports on workspaces that have sandboxes defined, and can only create imports from source views, source tables, Chorus views in a workspace, or .csv files from your local system.

Data can be imported as:

  • Into a new table: Creates a new table and imports the data into the new table. You can specify the number of rows that you want to import.
  • Into an existing table: Import a dataset into an existing sandbox table. You can truncate the data if necessary. Note that the table name must be specified from the drop-down menu.
  • From a .csv file from your local system as a table into the sandbox:
    • Click the Import File button, then select the file you want to import.
    • Specify a name for the file, then click Upload file.
    • Specify the type of your variables, a delimiter type, and the column names.

Import Now. The Import Now option in the right pane is only available for source table/views and Chorus views. Selecting Import Now will begin the import of the table immediately.

Scheduling an import

You can schedule an import of a table on a daily, weekly, or monthly basis. This automatically starts the process at the time you specified and imports the data in the destination file you also specified.

Adding a Sandbox

Create a sandbox so that you can access the full feature set associated with the Data tab.

Only the workspace owner can add a sandbox. Workspace members should contact the workspace owner to request this action.

  1. Hover over and then select the ? icon that appears next to the Import file button at the top left of the Workspace Data tab. The Add a Sandbox dialog opens.
  2. Select either:
  • within an instance: Using the drop-down options, select an instance, database, and schema to associate with your sandbox. Each user can only be associated with a single sandbox.
  • as a standalone: A self-contained Greenplum instance can be provisioned on the Chorus Virtualization Service within minutes. You need to specify an account and password to this instance that will be used as a shared account with super-user privileges.

When you create a new instance on the Chorus Virtualization Service, you can specify the instance size and also select an appropriate resource allocation template (vCPU and memory to do it).

You can also add a sandbox via the workspace Summary tab. See Summary for details.

After you have added a sandbox, all the datasets in the database/schema are automatically associated with the workspace and are shown in the Data tab.
