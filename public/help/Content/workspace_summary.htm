Summary

Summary is the first tab in the workspace. The Summary tab presents information about the recent activity associated with that workspace.

Features of the Summary tab

  • Viewing your Summary tab
  • Editing a workspace (all users)
  • Editing your workspace (Administrator or owner only)

Viewing your Summary tab

  • Displays all the activity associated with the workspace or objects belonging to the workspace. The default view is Show:All Activity.This displays all the activity that has occurred in the workspaces such as names of files that have been added, the name of the source table and the name of the destination table for the data, scheduled imports, and any associated comments and insights.
  • You can switch to Insights filter the activity stream to only show insights made on the workspace or objects belonging to the workspace.

Editing a workspace (all workspace members)

Note non-workspace members cannot edit the workspace, and will only ever see a read-only Workspace Settings dialog.

The following is a list of tasks that all workspace members can perform.

  • Edit Workspace: Select this option to open the Workspace Settings dialog. You can only edit the workspace name and summary.
  • Add a note: Select this option to comment on this workspace. You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.
  • Add an insight: Select this option to add an insight about this workspace. Insights are useful for all Chorus users, and appear in the workspace activity stream and the workspace member's Home page activity stream. If the insight is published, it also appears on all Chorus users' Home page activity streams. You can add a desktop file, work file, or dataset to add support your insight. You can also remove any attachments and notify selected members on your project team about your activity.

Editing your workspace (Administrator or owner only)

The following is an list of additional tasks that you can perform if you own the workspace or are a Chorus Administrator.

  • With a workspace selected, click Edit Workspace in the right pane to open the Workspace Settings dialog. You can edit the following options:
    • Workspace Name: You may want to edit the name of a workspace for a variety of reasons, such as providing clarity, or labeling your workspaces. You can also choose to make your workspace public or opt to keep it private. Only workspace members can view private workspaces. Non-workspace members can browse public workspaces in read-only mode.
    • Summary: Add a short description about the workspace. You can format your summary using the text options.
    • Workspace Image: Select a PNG, GIF, or JPG image and click Add located below the Workspace Image field. The Add option switches to Change, once you have uploaded an image.
    • Sandbox location: Displays the sandbox location. If you have not added a sandbox, this field displays the location as None.
    • Change owner: You can delegate ownership of one of your workspaces to another member of your team.
    • Workspace status: Either Active or Archived. Once you archive a workspace it is only available in read-only mode.
  • Delete this Workspace: Performing this action cannot be undone. Also, all associated imports will be unscheduled.
  • Add or Edit Member: Editing the list of members list helps with the collaborative process. They can add files, import data, and receive messages about the workspace.
  • Add a Sandbox: This option is available only if you own the workspace. This option opens the Add a Sandbox dialog, here, select either:
    • within an instance: Using the dropdown options, select an instance, database, and schema to associate with your sandbox. Each workspace can only be associated with a single sandbox.
    • as a standalone: A self-contained Greenplum instance can be provisioned on the Chorus Virtualization Service within minutes. You need to specify an account and password to this instance that will be used as a shared account with super-user privileges. When you create a new instance on the Chorus Virtualization Service, you can specify the instance size and also select an appropriate resource allocation template (vCPU and memory to do it).

You can also add a sandbox via the workspace Data tab. See Data for details.

Note: If you do not add a sandbox, then you may not have access to all the features offered on the Data tab.
