Summary

Summary is the first tab in the workspace folder. The Summary tab presents information about the recent activity associated with that workspace.

Features of the Summary tab

  • Viewing your Summary tab
  • Editing a workspace (all users)
  • Editing your workspace (Administrator or owner only)

Viewing your Summary tab

  • Displays all the activity associated with the workspace, the default view is Show:All Activity.You can view all the activity that has occurred in the workspaces such as names of files that have been added, the name of the source table and the name of the destination table for the data, scheduled imports, and any associated comments.
  • You can switch to Insights to read general comments and insights from other Chorus members. These insights may be with reference to other workspaces.

Editing a workspace (all users)

The following is a list of tasks that all users can perform.

  • Workspace Settings brings up the Workspace Settings dialog. You can only edit the following options:
  • Add a note to comment on a particular workspace.You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.
  • Add an insight is useful for all Chorus users,and appears on Home dashboard. You can add a desktop file, work file, or dataset to add support your insight. You can also remove any attachments and notify selected members on your project team about your activity.

Editing your workspace (Administrator or owner only)

The following is an list of additional tasks that you can perform if you own the workspace or are a Chorus Administrator.

  • Edit Workspace to see the Workspace Settings dialog. You can edit the following options:
    • Workspace Name: You may want to edit the name of a workspace for a variety of reasons, such as providing clarity, or labeling your workspaces. You can also make your workspace private by unchecking the Publicly available option.
    • Summary: Add a short description about the workspace. You can format your summary using the text options.
    • Workspace Image: Select a PNG, GIF, or JPG image and click Add located below the Workspace Image field. The Add option switches to Change, once you have uploaded an image.
    • Sandbox location: Displays the sandbox location. If you have not created a sandbox, this field displays the location as None.
    • Change owner: You can delegate ownership of one of your workspaces to another member of your team. If you are an admin, you may have set up workspaces for the members of the team and need to give them access.
    • Workspace status: Once you archive a workspace it is only available in read-only mode.
  • Delete this Workspace this action cannot be undone.
  • Add or Edit Member to the workspace to help with the collaborative process. They can add files, import data, and receive messages about the workspace.
  • Add a Sandbox is available only if you own the workspace. Brings up the Add a Sandbox dialog. Using drop-down options, you can select an instance, database, and schema to associate with your sandbox. Each user can only be associated with a single sandbox.
