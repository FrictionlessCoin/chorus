Instances

The Instances page allows you to explore and edit the major areas associated with instances. The following topic presents a list of features that best describe the information you can access and tasks you can perform from this page.

Features of the Instances page

  • Exploring instances
  • Administering an instance
  • Working with instances
  • Viewing workspaces

Exploring instances

  • Browse the list presented in the left pane. You will see instances grouped under Greenplum, Hadoop, and other.
  • Highlight an instance to view information about it in the right pane.
    • If you are a Chorus administrator, you can Editinstance permissions, Edit Instance, View Workspaces, and Add a note. You can Comment on the activity, and Promote to Insight.
    • If you are a User you can Edit Credentials, Remove Credentials,View Workspaces, and Add a note. You can Comment on the activity, and Promote to Insight.

Administering an instance

  1. Edit: Select this option to bring up the Instance Permissions dialog. You can only edit the instance if you are a member of the workspace with which it is associated.
  2. Add Account: Select this option to grant a user permissions to the instance. Select the user from the drop down list and provide your username and password.
  3. Edit: Use this option to change the ownership of the instance.
  4. Edit Instance: Use this option to make configuration changes.

Working with instances

  • Add Instance to bring up the Create New Instance dialog. You can select one of the following:

    • Register an existing Greenplum Database instance

    • Register an existing Hadoop Distributed File System (HDFS): selecting this option allows you to browse the file server for certain text files. They can create an external table to the file using the Greenplum connector, gphdfs, on a workspace's sandbox schema. You would need to specify the data types and the file delimiter so that you can create an external table for it.
  • Edit Credentials: Presents the Edit Instance Credentials dialog. Enter your username and password to access the instance. You can only do if your administrator grants you those privileges.
  • Remove Credentials: Presents you with the option of removing your credentials from an instance. You will be required to do so,before you can delete your account.

Viewing Workspaces

  • View Workspaces: Displays the number of workspaces that you can view. Presents the Chorus Usage by Workspace: dialog. This dialog shows information about the location and the owner of workspaces.
  • Add a note: Provides you with the option to comment on a particular instance.You have the option to add a desktop file, work file, or dataset to add support your note. You can also remove any attachments and notify selected members on your project team about your activity.
