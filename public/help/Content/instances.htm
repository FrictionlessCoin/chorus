Instances

The Instances page allows you to explore and edit the major areas associated with instances. The following topic presents a list of features that best describe the information you can access and tasks you can perform from this page.

Features of the Instances page

  • Exploring instances
  • Administering an instance
  • Working with instances
  • Viewing associated workspaces
  • Exploring data

Exploring instances

  • Browse the list displayed in the left pane. You will see instances grouped under Registered Greenplum Instances, Registered Hadoop Instances, and Other Instances.
  • Highlight an instance to view information about it in the right pane.
    • For Greenplum Database instances, if you are a Chorus administrator or instance owner, you can Editinstance permissions, Edit Instance, View Workspaces, and Add a note. You can Comment on the activity, and Promote to Insight. For Hadoop instances
    • If you are a User you can Edit Credentials, Remove Credentials,View Workspaces, and Add a note. You can Comment on the activity, and Promote to Insight.

Administering an instance

  1. Setting/editing permissions

Note that the following information about setting/editing permissions only applies to Greenplum Database instances, not Hadoop instances.

  • Edit: Select this option from the Instances page to open the Instance Permissions dialog. If you are a Chorus administrator or the instance owner, you can change the instance permissions here to be either a Shared single account, or an Individual account.
    • In shared account mode, Chorus will use this single Greenplum Database username and password for all Chorus users. This account should have the lowest level of permissions available. This account is also used by Chorus to index the instance for search. In shared mode, only Administrators and instance owners can edit this credential.
    • In Individual account mode, each Chorus user must provide their own username and password for Chorus to use to connect to the Greenplum Database instance. The instance owner's account will be used by Chorus to index the instance for search. In this mode, Chorus users can add/edit/remove their own credentials to the instance.
  • Add Account: Select this option to grant a user permissions to the instance. Select the user from the drop down list and provide your username and password. Note that this functionality is only available if you are in Individual Account mode.
  • Edit: Select this option from within the Instance Permissions change the user name and password that are used to connect to the instance.
  • Owner: Select this option to transfer ownership of the instance to another user. If you are in Shared Account Mode, you can transfer ownership to any Chorus user. If you are in Individual Account mode, you can only transfer ownership to members who have their account mapping for the instance added.

2. Configuring the instance

  • Edit Instance: Select this option from the Instances page to make configuration changes such as instance name, description, host and ports.

Working with instances

  • Click Add Instance on the Instances page to open the Create New Instance dialog. You can select one of the following:
    • Register an existing Greenplum Database instance.
    • Create a new Greenplum Database instance.
      • A self-contained Greenplum instance can quickly be provisioned on the Chorus Virtualization Service. You need to specify an account and password to this instance that will be used as a shared account with super-user privileges. When you create a new instance on Chorus Virtualization Service, you can also specify the instance size and select an appropriate resource allocation template (vCPU and memory) for it.
    • Register an existing Hadoop Distributed File System (HDFS). Selecting this option allows you to browse the file server for certain text files. You can create an external table to the file using the Greenplum connector, gphdfs, on a workspace's sandbox schema. You will need to specify the data types and the file delimiter so that you can create an external table for it.
  • Add/Edit Credentials: This opens the Add/Edit Instance Credentials dialog. Enter the username and password to access the instance. This option is only visible for non-administrative users who are also not the instance owner for a Greenplum Database that is in Individual account mode.
  • Remove Credentials: Allows you to removes your credentials from Chorus. This option is only visible for non-administrative users who are also not the instance owner for a Greenplum Database that is in Individual account mode.

Viewing Workspaces

  • The View Workspaces link displays the number of workspaces with sandboxes on this instance. Clicking this link opens the Chorus Usage by Workspace dialog. This dialog displays information about how much disk space the sandbox schemas are using and if the space used exceeds the recommended disk space for a sandbox schema. The recommended disk space can be configured by the Chorus Administrator. See the Chorus Installation Guide for details.
  • The Add a note link provides you with the option to comment on a particular instance. You have the option to add a desktop file to support your note. You can send the note as a notification to any Chorus user.

Exploring Data

You can explore the data on an instance that you have registered by clicking on the instance on the Instance page.

Hadoop: When you click on a Hadoop instance, its files and directories are displayed. The following actions are then available: drilling down into directories, previewing files, adding notes to files, creating external tables to a file on a selected workspace's sandbox, and if the workspace's sandbox instance is Greenplum Database version 4.2 or above, you can specify a directory or set of files in a directory as an external table on the selected workspace's sandbox.

Greenplum: When you click on a Greenplum Database instance, the databases and schemas under it are displayed. The following actions are then available:

  • Adding a note. A note can be added to the table/view in a schema. You can support your note with attachments from the desktop only. You can send a notification to other Chorus users about the note.
  • Attaching table(s)/view(s) to a workspace. You can attach a table/view to a workspace by selecting the table and clicking on Associate dataset with a workspace in the right pane. To attach multiple tables/views, check the box on the table to the left of the table name and click on Associate with a workspace in the right pane.
  • Previewing data. You can preview a table's content and data by clicking on the Preview Data option. This queries the table and returns the first 100 rows in a table. You also have the option to download to your desktop a specific number of rows, or to download all the rows.
  • Visualizing data. You can create a visual representation of your data by clicking on a table, then specifying Visualize. You can also add data filters. The types of charts you can use to visualize your data are as follows:
    • Frequency chart
    • Histogram (currently only supports numerical data)
    • Heatmap (currently only supports numerical data)
    • Time Series
    • Boxplot
  • Running Analyze. You can click Run analyze to collect statistics about the contents of a table. The results of this analyze are stored in the system table pg_statistic. The query planner will subsequently use these statistics to help determine the most efficient execution plans for queries. These statistics, including row count, common values, and so on, can be viewed from the Information sub-tab for the table. Note: Running analyze on large tables can significantly slow down queries, so should be used with caution.
